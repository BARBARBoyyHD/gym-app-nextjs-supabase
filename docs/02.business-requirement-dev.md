## Gym Membership & Fitness Course System



## 1. Overview

Aplikasi ini merupakan platform manajemen member gym dan sistem pembelajaran fitness berbasis video.  
Platform menyediakan:

- Manajemen member dan status membership  
- Paket membership dengan batas waktu  
- Akses kursus dan materi video fitness  
- Dashboard admin untuk pengelolaan konten

Admin menggunakan **Supabase Auth** untuk otentikasi, sedangkan data member dicatat pada tabel `users` di PostgreSQL.

---

## 2. Goals

- Menyediakan sistem pencatatan member yang sederhana dan scalable  
- Memungkinkan admin mengelola paket membership dan kursus  
- Member dapat mengakses kursus video sesuai status membership  
- Menghasilkan sistem demo yang terlihat production-ready

---

## 3. High-Level Features

### 3.1 Admin (Supabase Auth)
- Login via Supabase
- CRUD Membership Plans
- CRUD Courses (link video embed)
- Melihat daftar member dan status membership

### 3.2 Member (Database User)
- Melihat informasi membership
- Melihat daftar kursus
- Menonton video kursus berbentuk embed
- Melihat detail profil (basic data)

---

## 4. Functional Requirements

### 4.1 Member Management
- Input data member: nama, kontak, tanggal bergabung
- Mencatat status membership: active / expired
- Menyimpan riwayat membership per member

### 4.2 Membership Plans
- Admin dapat membuat paket (nama, harga, durasi)
- Sistem menghitung tanggal berakhir membership otomatis

### 4.3 Course Management
- Admin dapat menambah kursus:
  - Judul
  - Deskripsi
  - Kategori
  - Video embed URL
  - Thumbnail URL

### 4.4 Video Access
- Member dapat membuka halaman kursus
- Video ditampilkan menggunakan link embed (YouTube / Vimeo / Supabase Storage)

### 4.5 Access Control
- Member hanya dapat mengakses kursus jika status membership aktif
- Admin memiliki akses ke menu pengelolaan

---

## 5. Non-Functional Requirements

### 5.1 Security
- Admin hanya bisa diakses melalui Supabase Auth
- Data member menggunakan RLS (Row-Level Security)
- Validasi input embed URL (whitelist domain)

### 5.2 Performance
- List courses harus dapat diload dalam < 1.5 detik
- Pagination untuk kursus dan member

### 5.3 Maintainability
- Kode mengikuti struktur proyek berbasis modul
- CRUD menggunakan pattern reusable hooks / API functions

### 5.4 Scalability
- Database mendukung pertambahan data tanpa modifikasi struktur
- Penyimpanan video menggunakan external embed untuk menekan biaya

## 7. Business Rules

1. Member hanya bisa memiliki **satu status membership aktif** pada satu waktu.
2. Membership otomatis berubah menjadi **expired** jika `end_date < current_date`.
3. Kursus hanya dapat dilihat member dengan status **active**.
4. Video harus berbentuk **embed URL**, bukan file upload tinggi bandwidth.
5. Admin harus login melalui **Supabase Auth** untuk akses halaman manajemen.

## 9. Risks & Mitigation

| Risiko | Mitigasi |
|-------|---------|
| Video bandwidth tinggi | Gunakan embed YouTube/Vimeo |
| Private access course bocor | Validasi membership + policy RLS |
| Expired membership tidak update | Cron job / scheduled function di Supabase |
| Input embed URL sembarangan | Whitelist domain (YouTube, Vimeo, Supabase) |
