## 8. Technical Architecture (High-Level)

- **Frontend:** Next.js (App Router, Tailwind, shadcn UI)
- **Backend:** Supabase Postgres
- **Auth:** Supabase Auth (admin only)
- **Storage:** Supabase Storage / YouTube embed
- **Hosting:** Vercel
- **Data Access:** Supabase JS client

**Access Control Design:**
- Admin pages: server-only actions + Supabase Auth session
- Member pages: public read + database filtered by membership status

---

## 9. Risk Management

### Risks
- Salah konfigurasi RLS menyebabkan data bocor
- Embed video salah domain / tidak aman
- Deployment gagal karena env var salah
- Penggunaan free tier supabase berpotensi limit

### Mitigation Actions
- Gunakan **RLS preset templates** untuk CRUD
- Validasi only:
  - youtube.com  
  - vimeo.com  
  - *.supabase.co/storage  
- Buat `.env.example`  
- Gunakan embed video, bukan file upload untuk optimisasi bandwidth

## 10. Quality Assurance & Testing

### Test Scenarios
- CRUD testing untuk members, plans, courses

- Status membership otomatis berubah menjadi expired jika melewati tanggal
